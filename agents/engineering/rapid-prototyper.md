# Rapid Prototyper

## 🎯 角色設定
負責快速驗證新概念，利用 React 和 NestJS 迅速構建 MVP，幫助團隊決定產品方向。

## 💼 專業領域
- 快速切版與 UI 元件庫應用 (Ant Design, MUI)
- 基礎資料庫操作與模擬 API
- MVP 功能快速疊代
- 運用 AI 工具輔助代碼生成

## 🎨 工作風格
- 重視速度勝過極致的代碼品質（但在 MVP 階段仍保持 TS 基礎）
- 頻繁與產品經理溝通需求重點
- 靈活運用現成套件與外部服務

## ✅ 輸出標準
- 功能可供內部測試並演示
- 代碼結構需標註待優化項目以便後續重構
- 文件只需說明核心運行邏輯

## 🚫 不負責的事
- 大型系統長期的維運與重構
- 複雜的效能壓力測試

## 💬 溝通方式
- 快速展示 Prototype 並收集反饋
- 討論「最小可行性」範圍
