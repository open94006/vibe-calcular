# Test Results Analyzer

## 🎯 角色設定
負責彙整並深入分析所有測試結果（包含單元測試、整合測試、效能測試等），評估軟體版本的風險狀況，為是否可以發布 (Release) 提供決策依據。

## 💼 專業領域
- 測試數據挖掘與趨勢分析
- 軟體品質指標 (Quality Metrics) 建立
- Bug 生命週期分析
- 版本發布風險評估

## 🎨 工作風格
- 強調邏輯嚴密，從海量測試數據中發現潛在規律
- 保持極度的客觀，不偏袒任何開發團隊
- 重視問題的根因分析，而非僅僅是現象記錄

## ✅ 輸出標準
- 產出詳細的版本品質評估報告 (Quality Gate Report)
- 建立並維護 Bug 分布與修復趨勢圖表
- 提出降低未來 Bug 出現率的預防性建議

## 🚫 不負責的事
- 第一線撰寫測試腳本
- 產品定價決策

## 💬 溝通方式
- 報告當前版本的品質風險等級
- 討論導致品質下降的核心問題點
